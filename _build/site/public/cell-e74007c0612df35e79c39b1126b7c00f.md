# Cell Statistics and Map Algebra

Map algebra is a fundamental concept in raster GIS, involving applying mathematical operations to one or more raster layers. In this technique, raster layers can be treated like variables in algebra to apply arithmetic, including addition, subtraction, division, averaging, etc.

## Cell Statistics

The [Cell Statistics tool](https://pro.arcgis.com/en/pro-app/3.3/tool-reference/spatial-analyst/cell-statistics.htm) in ArcGIS Pro or in QGIS under [Raster analysis](https://docs.qgis.org/3.44/en/docs/user_manual/processing_algs/qgis/rasteranalysis.html) calculates a per cell statistic across multiple rasters. When these rasters overlap, a statistic is applied to them to produce an output raster. When the rasters do not overlap, the rasters can be combined to increase the extent of data; for example, when combining several tiles of data (e.g., digital elevation models).

## Raster Calculator

The Cell Statistics tool has a limited number of mathematical operations. The [Raster Calculator tool](https://pro.arcgis.com/en/pro-app/3.3/tool-reference/spatial-analyst/raster-calculator.htm) in ArcGIS Pro or QGIS under [Raster analysis](https://docs.qgis.org/3.44/en/docs/user_manual/processing_algs/qgis/rasteranalysis.html) provides more flexibility in defining such operations.

We can also set conditional statements. The following operation defines a conditional statement that where values in the DEM are less than 300, they should be set to 0, otherwise all other values should keep the same value in the raster. In ArcGIS Pro's Raster Calculator, run the following:

```
Con("dem.tif" < 300, 0, "dem.tif")
```

In QGIS, use if( instead of Con(

```
if("dem.tif" < 300, 0, "dem.tif") 
```

We can also use a conditional statement to replace NoData values with values from another raster. In the following example, we are filling the voids of a 1-arc second SRTM DEM with the values from a void-filled 3-arc second SRTM DEM. If we set the cell size under Environments to Minimum of Inputs, the output raster will be resampled to 1-arc second resolution.

In ArcGIS Pro:

```
Con(IsNull("1arc_dem.tif") , "3arc_dem.tif", "1arc_dem.tif")
```

In QGIS, you first need to run the Fill NoData cells tool under [Raster tools](https://docs.qgis.org/3.44/en/docs/user_manual/processing_algs/qgis/rastertools.html) to define a NoData value, for example, -9999. Then in Raster Calculator copy the following expression:

```
if("1arc_dem.tif" = -9999, "3arc_dem.tif", "1arc_dem.tif")
```

Stated in plain English, this conditional statement can be read as "When the 1-arc second DEM has NoData, replace these values with those from the 3-arc second DEM, otherwise keep the 1-arc second DEM."

Note that QGIS has several Raster Calculator tools that run the same processes but use different syntax. If running Raster Calculator from Raster analysis, you must first load the input layers, then you can define an expression using these input layers and the appropriate operations. Another Raster Calculator tool is available under GDAL. Here, you can define any input rasters as A, B, C, D, E, or F. Then the expression can be defined using gdalnumeric syntax, for example, A-B will subtract the second raster from the first, and A*2 will multiple all values in the raster by a factor of 2. An additional Raster Calculator is available under Raster -> Raster Calculator, which does not require the inputs to be defined; instead, all rasters under Layers are available. (The latter is my preferred version of Raster Calculator in QGIS.)
