In QGIS, under Symbology in Layer Properties, change the Interpolation to Discrete, adjust the Mode (Continuous, Equal Interval, or Quantile), and set the number of Classes.

These visualizations are only temporary, however, and the actual raster values remain unchanged.

## Reclassify

We might want to make such a classification permanent by generating a new raster grid where each cell now has a value representing a category rather than an elevation. To do so, we can use the [Reclassify tool](https://pro.arcgis.com/en/pro-app/3.4/tool-reference/spatial-analyst/reclassify.htm) in ArcGIS Pro or Reclassify by table under [Raster analysis](https://docs.qgis.org/3.44/en/docs/user_manual/processing_algs/qgis/rasteranalysis.html) in QGIS. In ArcGIS, after opening the Reclassify tool and loading the appropriate input raster surface layer, the tool will automatically populate the reclassification table with the values from the symbology. Alternatively, if the raster layer is currently set to a stretch symbology, we can click the Classify button to define a classification.

In QGIS, in the Reclassify by table tool, create a Reclassification table by adding the desired number of rows to establish ranges of values (Minimum and Maximum) to be reclassified to a single Value. These value, for example, could be copied manually from the Symbology defined in Layer Properties. Select an output data type. The tool automatically selects Float32, which unlike the tool in ArcGIS, allows for negative and decimal values. However, reclassification should generally use integer values; thus, Byte (8-bit unsigned), Int8 (8-bit signed), Int16 (16-bit signed), or UInt16 (16-bit unsigned) are usually better options. If changing the Output data type, the Output NoData value must also be changed. The default value is -9999, but this value will have to be changed depending on the range of your chosen output data type. The default range value is usually appropriate; min < value <= max will include your maximum value but exclude your minimum. The check box next to Use NoData when no range matches value should be checked.

```{image} /images/reclassifytool.jpg
:alt: Reclassify Tool
:class: bg-primary mb-1
:width: 80%

```

According to this table, all values in the range from Start to End will be reclassified to the value under the New column. NODATA will remain unchanged. The output raster will look similar to the classified output, but now the range of cell values is only 1 to 5, rather than -22 to 1925.

The Reclassify tool is most useful when establishing inputs of equal weights when developing models. For example, different input variables (e.g., elevation, soil type, precipitation) can be scaled between 1 to 10 to assist with comparison.

In QGIS, to reclassify NoData pixels, you first need to run Fill NoData cells under [Raster tools](https://docs.qgis.org/3.44/en/docs/user_manual/processing_algs/qgis/rastertools.html). Select a Fill value; this value will be assigned to NoData pixels. Be sure to select a value outside the range of your data. Then run Reclassify by table with a reclassification table with two rows: one with a range where the minimum and maximum are exactly equal to the value you assigned NoData in the Fill NoData cells tool, and the second with a range encompassing all other values in the input raster. Choose the values to be assigned with the reclassification. Then, set the Output NoData value to match the reclassified value assigned to the full range of your data.

Note that the Reclassify method only allows for integer outputs (although values can be positive or negative) in ArcGIS Pro.

## Slice

The [Slice tool](https://pro.arcgis.com/en/pro-app/3.3/tool-reference/spatial-analyst/slice.htm) in ArcGIS Pro is similar to Reclassify, but the specific ranges of values for the reclassification do not need to be defined. Instead, the required parameters are the number of slices (or output zones), the starting value for the output, an optional value to reclassify NODATA, and a slice method. The resulting output will be similar to the Reclassify tool.

The advantage of the Slice tool is that a reclassification method can be assigned to multiple input rasters that may have very different ranges. This approach is most useful when automating processes, where several rasters can be input with the same parameters to create scaled outputs.

Currently, QGIS only supports reclassifying rasters based on specific values, rather than reclassification methods.
